# ⏱️ Analytics in 60 Seconds (Ad-hoc Dictionary)

Small, sharp entries — each one explainable in under a minute. This is my personal quick-reference dictionary, published so others can use it too.

## Sections

| Folder | Contents |
|--------|----------|
| `sql-tricks/` | One trick per file: e.g., `QUALIFY vs subquery`, `anti-join patterns`, `deduplicating with ROW_NUMBER` |
| `excel-formulas/` | Real-world formula patterns: XLOOKUP chains, SUMPRODUCT conditions, dynamic arrays |
| `business-metrics/` | Metric definitions that actually matter: what it measures, formula, when it misleads |

## Entry format

Every entry answers three things, fast:

```markdown
# Metric/Trick Name

**What it is** — one sentence.
**How to do/calculate it** — the formula or snippet.
**When it bites you** — the common mistake or misleading edge case.
```

## Why 60 seconds?

Because in real work, most analytics questions are ad-hoc: someone leans over and asks *"quick — what's our churn rate formula?"* An analyst's value is often measured in how fast and how clearly they can answer.
