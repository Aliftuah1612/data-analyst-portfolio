# 🏃 Monthly Skill Sprint

A structured, month-by-month learning path. One skill per sprint, with notes and evidence of practice committed here as I go.

## The Track

| Sprint | Skill | Focus |
|--------|-------|-------|
| 1 | **SQL** | From solid fundamentals to window functions & query optimization |
| 2 | **Excel** | Lookup mastery, pivot modeling, dynamic arrays, what-if analysis |
| 3 | **Power BI** | Data modeling, DAX, report design for stakeholders |
| 4 | **Python** | pandas for analysis, automation of repetitive analyst work |
| 5 | **Statistics** | Distributions, hypothesis testing, confidence — knowing when a number means something |
| 6 | **Data Visualization** | Chart choice, perception principles, decluttering |
| 7 | **Storytelling** | Structuring findings into narratives that drive decisions |
| 8 | **AI for Analytics** | Using LLMs responsibly: faster EDA, query drafting, sanity-checking |

## Sprint structure

```
sprint-XX-skill-name/
├── README.md        ← goals, resources used, weekly log
├── notes/           ← distilled learning notes (not copy-paste — my own words)
└── practice/        ← exercises, mini-projects, before/after examples
```

## Rules of the sprint

1. **Output over input** — every sprint ends with something built, not just something read.
2. **Notes in my own words** — if I can't explain it simply, I haven't learned it.
3. **Connect to the portfolio** — each sprint should feed the [monthly project](../05-one-project-per-month) or the [60-seconds dictionary](../03-analytics-in-60-seconds).
