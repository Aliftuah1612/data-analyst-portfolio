# 📈 Dashboard Collection

A collection of dashboards, each built on a **different dataset**, so recruiters can explore range: different industries, different metrics, different design decisions.

## How each dashboard is documented

Every dashboard gets its own folder containing:

```
dashboard-name/
├── README.md        ← the story: dataset, audience, key questions, design decisions
├── screenshots/     ← preview images (recruiters shouldn't need to install anything)
├── files/           ← .pbix / Excel / Tableau workbook / link to live version
└── data/            ← dataset source or sample (when license allows)
```

## Design principles I follow

1. **One screen, one decision** — a dashboard should help someone decide something, not display everything.
2. **Glanceable first, explorable second** — KPIs up top, drill-downs below.
3. **Context beats numbers** — every metric shown against a target, trend, or comparison.
